# 双臂机器人位置坐标确认表

> **目的**：在正式使用前，需现场测量并填入所有标注 `【待确认】` 的坐标/角度值。  
> **坐标系**：所有末端执行器位姿均基于 `base_link`（机器人底座中心）。  
> **单位**：xyz 单位为 **米（m）**，rpy 单位为 **弧度（rad）**（π ≈ 3.14159，π/2 ≈ 1.5708）。

## 六、如何测量实际坐标（操作步骤）

### 方法 A：RViz 拖拽读取

```bash
# 1. 启动 MoveIt + RViz（含 fake hardware）
ros2 launch openarm_bimanual_moveit_config demo.launch.py \
    use_fake_hardware:=true

# 2. 在 RViz 中：
#    - 找到 MotionPlanning 面板
#    - 拖动末端执行器交互标记（彩色箭头）到目标位置
#    - 在 "Query Goal State" 中读取 XYZ + RPY

# 3. 用 tf2 读取当前末端位置（先让机械臂运动到目标位置）
ros2 run tf2_ros tf2_echo base_link openarm_right_hand_tcp
ros2 run tf2_ros tf2_echo base_link openarm_left_hand_tcp
```

### 方法 B：关节空间 → 笛卡尔空间换算

```bash
# 查看当前关节角度
ros2 topic echo /joint_states

# 用 MoveIt FK 服务计算末端位姿（需 skill_server_node 运行中）
# 或直接在 RViz 中 "Update" 后读取 End Effector 位姿
```

### 方法 C：真实硬件示教

## 七、确认完成后如何填入配置

### 更新 pick_place 默认坐标

编辑 `src/openarm_manipulator_board/config/manipulator_board.yaml`：

```yaml
use_default_poses: true

# 右臂（运动规划坐标系：base_link）
default_grasp_xyz: [0.42, -0.15, 0.25]   # 替换为实测值
default_grasp_rpy: [0.0, 1.5708, 0.0]

default_place_xyz: [0.30, -0.30, 0.25]   # 替换为实测值
default_place_rpy: [0.0, 1.5708, 0.0]
```

> **注意**：修改 yaml 后重启 `manipulator_board.launch.py` 即可，无需重新编译。

### 更新 Home 位置（需重新编译）

编辑 `src/openarm_ros2/openarm_bimanual_moveit_config/config/openarm_bimanual.srdf`，  
修改对应 `group_state name="home"` 中各关节的 `value`，然后：

```bash
colcon build --packages-select openarm_bimanual_moveit_config
source install/setup.bash
```
