# 机械臂调试说明（双臂）

> **适用场景**：在本机（板卡）上完成双臂「固定位置抓取 → 放置」功能的逐步验证。  
> **前提**：已完成 `colcon build`，并 `source install/setup.bash`。  
> **注意**：本机器人为**双臂**，左臂（`left`）和右臂（`right`）均有独立的规划组和夹爪，  
> `pick_place` 每次只控制**一只手**，`home` 可同时控制双臂。

---

## 第一部分：双臂坐标确定

所有位姿均以 **`base_link`（机器人底座中心）** 为参考系，单位：米 / 弧度。

### 坐标系参考

```
         +x（前方）
          ↑
          |
右臂侧←--base_link--→左臂侧
   −y         +y
          |
          ↓

+z = 上方
```

> 通常：右臂在 y 轴负方向（`y < 0`），左臂在 y 轴正方向（`y > 0`）。  
> 以实际 URDF 为准，可通过 RViz 确认。

### 方法 A：RViz 拖动交互标记读坐标（推荐）

1. 启动 MoveIt + RViz（见第二部分 Step 1）。
2. 在 RViz 左侧 `MotionPlanning` 面板，**分别**切换 Planning Group：
   - 右臂：`right_arm`
   - 左臂：`left_arm`
3. 拖动交互标记到目标位置，点击 `Plan`，读取末端 `xyz`。
4. 记录左、右臂各自的抓取点和放置点坐标。

### 方法 B：tf2 读取当前末端位置

将机械臂手动推到目标位置后执行：

```bash
# 右臂末端
ros2 run tf2_ros tf2_echo base_link right_ee_link

# 左臂末端
ros2 run tf2_ros tf2_echo base_link left_ee_link
```

### 填写配置文件

`manipulator_board.yaml` 的默认点位只有一组（用于 `default_arm` 指定的那只手）。  
如果要**左右两臂都用固定点位**，命令中须显式传入 `arm` 字段和对应坐标（见命令 5B/5C）。

编辑 `src/openarm_manipulator_board/config/manipulator_board.yaml`：

```yaml
default_arm: "right"       # 命令未指定 arm 时默认使用右臂
use_default_poses: true

# 右臂默认抓取/放置点（default_arm=right 时生效）
default_grasp_xyz: [0.42,  -0.10, 0.20]  # TODO: 右臂实测抓取点
default_grasp_rpy: [0.0,    1.5708, 0.0]
default_place_xyz: [0.30,  -0.30, 0.20]  # TODO: 右臂实测放置点
default_place_rpy: [0.0,    1.5708, 0.0]
```

> 左臂的固定点位在调试命令中直接写入 `params`（见命令 5C），不占 YAML 的默认位置。

修改后重新构建：

```bash
colcon build --packages-select openarm_manipulator_board && source install/setup.bash
```

### 末端姿态 rpy 参考

| 末端姿态 | rpy 参考值 | 适用场景 |
|----------|------------|----------|
| 竖直向下 | `[0.0, 1.5708, 0.0]` | 桌面抓取 |
| 水平朝前 | `[0.0, 0.0, 0.0]` | 侧面夹取 |
| 倾斜 45° | `[0.0, 0.7854, 0.0]` | 斜面抓取 |

## 第三部分：逐步发送命令验证

在**终端 E**（新建）执行，每次发完观察终端 D 的 status 变化。

> **幂等提示**：每条命令的 `request_id` 必须**唯一**，重复的 id 会被缓存命中而不执行。  
> 建议按 `-001 -002 -003` 递增，或用 `$(date +%s)` 生成时间戳。

---

### 命令 1：解除安全互锁（最先发，必须）

```bash
ros2 topic pub --once /robot_arm/start std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_start\",\"enable\":true,\"mode\":\"manual\",\"request_id\":\"start-001\",\"timestamp\":1000}"}'
```

**期望 status：** `"state":"idle"`, `"ack":"done"`, `"error_code":0`

---

### 命令 2：查询状态

```bash
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"get_status\",\"request_id\":\"stat-001\",\"timestamp\":1000}"}'
```

**期望：** 立刻收到完整 status，`"state":"idle"`。

---

### 命令 3：双臂同时回 home（验证 MoveIt 规划）

```bash
# arm=both → 左右两臂同时回 home
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"home\",\"params\":{\"arm\":\"both\"},\"request_id\":\"home-001\",\"timestamp\":1000}"}'
```

**期望：** 左右两臂同时运动回 home，`"ack":"done"`。

---

### 命令 4：分别测试左右夹爪（无规划，最安全）

```bash
# 右夹爪打开
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"gripper\",\"params\":{\"action\":\"open\",\"arm\":\"right\"},\"request_id\":\"grip-r-open-001\",\"timestamp\":1000}"}'

# 右夹爪关闭
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"gripper\",\"params\":{\"action\":\"close\",\"arm\":\"right\"},\"request_id\":\"grip-r-close-001\",\"timestamp\":1000}"}'

# 左夹爪打开
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"gripper\",\"params\":{\"action\":\"open\",\"arm\":\"left\"},\"request_id\":\"grip-l-open-001\",\"timestamp\":1000}"}'

# 左夹爪关闭
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"gripper\",\"params\":{\"action\":\"close\",\"arm\":\"left\"},\"request_id\":\"grip-l-close-001\",\"timestamp\":1000}"}'
```

**期望：** 每次 status 经过 `accepted` → `running` → `done`，对应侧夹爪物理响应。

> **注意**：左右夹爪每次只能控制一个（技能层不支持双侧同时运行），等 `state:idle` 后再发下一条。

### 命令 5A：右臂 pick_place（用 YAML 默认坐标）

```bash
# 不带 arm 字段时自动使用 default_arm=right
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"pick_place\",\"request_id\":\"pp-r-001\",\"timestamp\":1000}"}'
```

```bash
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"pick_place\",\"params\":{\"arm\":\"right\",\"grasp_pose\":{\"xyz\":[0.42,-0.10,0.20],\"rpy\":[0.0,1.5708,0.0]},\"place_pose\":{\"xyz\":[0.30,-0.30,0.20],\"rpy\":[0.0,1.5708,0.0]},\"speed_scale\":0.05},\"request_id\":\"pp-r-002\",\"timestamp\":1000}"}'
```

---

### 命令 5C：左臂 pick_place（直接传坐标，speed=0.05 最慢）

```bash
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"pick_place\",\"params\":{\"arm\":\"left\",\"grasp_pose\":{\"xyz\":[0.42,0.10,0.20],\"rpy\":[0.0,1.5708,0.0]},\"place_pose\":{\"xyz\":[0.30,0.30,0.20],\"rpy\":[0.0,1.5708,0.0]},\"speed_scale\":0.05},\"request_id\":\"pp-l-001\",\"timestamp\":1000}"}'
```

> **双臂先后执行**：技能层同时只接受一个 Action goal（内部有忙标志）。  
> 左右臂任务需**串行**执行：等右臂 `ack=done` 后，再发左臂命令。

**pick_place 期望 status 变化：**

```
accepted
→ running  state=running  (pick.approach)
→ running  state=running  (pick.descend)
→ running  state=running  (pick.close_gripper)
→ running  state=running  (transport)
→ running  state=running  (place.descend)
→ running  state=running  (place.open_gripper)
→ done     state=idle     error_code=0
```

---

### 命令 6：紧急停止（任意时刻可用）

```bash
ros2 topic pub --once /robot_arm/command std_msgs/msg/String \
'{data: "{\"type\":\"manipulator_command\",\"command\":\"stop\",\"request_id\":\"stop-001\",\"timestamp\":1000}"}'
```

**期望：** 当前动作立即停止，status → `"state":"stopped"`。

## 第四部分：双臂调试注意事项

### 左右臂坐标独立设置

| 字段 | 右臂参考范围 | 左臂参考范围 |
|------|-------------|-------------|
| `y` 值 | 负数（`-0.05 ~ -0.60`） | 正数（`+0.05 ~ +0.60`） |
| `x` 值 | 前方正值（`0.20 ~ 0.65`） | 前方正值（`0.20 ~ 0.65`） |
| `z` 值 | 桌面高度（`0.05 ~ 0.50`） | 桌面高度（`0.05 ~ 0.50`） |

> 以上仅为参考，实际范围取决于机器人 URDF 和 `skills.yaml` 的 `workspace_radius_m`。

### 串行执行规则

```
右臂发出 pick_place  →  等 status.ack = "done"
                              ↓
                       左臂发出 pick_place  →  等 status.ack = "done"
```

若并发发出，第二条命令会收到 `error_code: 1003`（ARM_BUSY），需等待后重发。

### 紧急停止会同时停两臂

`stop` 命令调用 `/openarm/stop` 服务，该服务会同时停止左右臂所有运动组。

## 第五部分：错误排查速查表

| `error_code` | 名称 | 原因 | 处理方法 |
|---|---|---|---|
| `0` | OK | 成功 | — |
| `1001` | PARAM_ERROR | JSON 字段缺失或格式错误 | 检查 JSON，换新 `request_id` 重发 |
| `1002` | CMD_NOT_SUPPORTED | 未知命令字 / `pick`/`place` 单独使用 | 改用 `pick_place` |
| `1003` | ARM_BUSY | 另一侧手臂还在运动 | 等 `state:idle` 后重发 |
| `1004` | SAFETY_INTERLOCK | 未发 `start(enable=true)` | 先发命令 1 |
| `2001` | EXEC_TIMEOUT | skills 节点超时或未启动 | 检查终端 B |
| `2002` | HARDWARE_FAULT | MoveIt 规划/执行失败 | 检查坐标（见下方） |

### 坐标合法性验证

```bash
# 验证某个坐标是否在工作空间内
# workspace_radius_m=1.20, min_z=-0.10, max_z=1.50
python3 -c "
import math
# 修改 x,y,z 为要验证的坐标
x, y, z = 0.42, -0.10, 0.20
r = math.sqrt(x**2 + y**2)
ok = r < 1.20 and -0.10 <= z <= 1.50
print(f'x={x} y={y} z={z}  r={r:.3f}  合法={ok}')
"
```

### 其他常见问题

**规划失败但坐标看起来正确**  
→ 查看终端 B 日志中 `cartesian plan FAILED`，说明路径上有障碍或姿态不可达  
→ 尝试调整 `approach_offset_m`（当前默认 0.05 m），增大到 0.08~0.10

**幂等缓存命中（命令不执行）**  
→ 更换 `request_id`，例如用 `` `cmd-$(date +%s%N)` ``

**status 话题无数据**
```bash
ros2 node list | grep manipulator_board   # 确认节点在运行
ros2 topic list | grep robot_arm          # 确认话题存在
ros2 topic hz /robot_arm/status           # 确认频率约 5Hz
```

## 第六部分：速度参数

| 阶段 | `speed_scale` | 说明 |
|------|--------------|------|
| 初次调试 | `0.05` | 极慢，安全确认轨迹不碰撞 |
| 功能测试 | `0.10` | 默认值（每次抓放约 10~15 秒） |
| 正式使用 | `0.20~0.30` | 按现场需求调整 |

- **运输段**（搬运过程）速度由 `skills.yaml` 的 `transport_speed_scale: 0.30` 单独控制。
- **双臂同时 home** 时两臂都以 `home` 命令中 `speed_scale` 参数为准（默认 `0.30`）。

## 第七部分：双臂验证 checklist

```
□ Step 1：colcon build 通过，source install/setup.bash
□ Step 2：终端 A 启动 demo.launch.py，RViz 显示左右两臂
□ Step 3：终端 B 启动 skills，日志显示 "skill_server ready"
□ Step 4：终端 C 启动 manipulator_board，日志显示 "ready"
□ Step 5：终端 D echo /robot_arm/status，收到 state=stopped 帧

□ 命令 1：发 start(enable=true)  →  state=idle
□ 命令 2：发 get_status          →  收到完整 JSON
□ 命令 3：发 home(both)          →  左右两臂同时回 home

□ 命令 4a：右夹爪 open/close     →  右夹爪物理响应
□ 命令 4b：左夹爪 open/close     →  左夹爪物理响应

□ 命令 5B：右臂 pick_place(speed=0.05)  →  ack=done，全流程无碰撞
□ 命令 5C：左臂 pick_place(speed=0.05)  →  ack=done，全流程无碰撞

□ 命令 6：执行中发 stop  →  双臂立即停止，state=stopped
```
